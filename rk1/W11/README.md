# README: Внешнее произведение бинарных векторов

## Общее описание

Программа демонстрирует интерактивное редактирование двух бинарных векторов (горизонтального и вертикального) и синхронное вычисление матрицы их внешнего произведения. Каждый элемент векторов и матрицы представлен отдельным графическим окном X11. Цвет ячейки кодирует её значение: **серый** = 0, **зелёный** = 1.

Математически:
- Горизонтальный вектор **H** длины N (элементы 0/1)
- Вертикальный вектор **V** длины N (элементы 0/1)
- Матрица внешнего произведения **M** размера N×N, где `M[i][j] = V[i] AND H[j]` (логическое умножение).

Пользователь может инвертировать значения в векторах (клик мыши по ячейке, глобальные комбинации клавиш), после чего матрица мгновенно пересчитывается и перерисовывается.

Программа написана на языке **C** с использованием библиотеки **Xlib** (X Window System).

## Требования к системе

- Операционная система с X Window System (Linux, BSD, другие UNIX-подобные).
- Установленная библиотека X11 (разработочные файлы).
- Компилятор GCC.

## Компиляция и запуск

### Компиляция из одного файла (исходный вариант)

```bash
gcc -o outer_product_editor outer_product_editor.c -lX11 -Wall
```

### Компиляция раздельной сборки (три файла: head.h, logic.c, display.c)

```bash
gcc -o outer_product_editor display.c logic.c -lX11 -Wall
```

### Запуск

```bash
./outer_product_editor <размерность_векторов>
```

Пример:

```bash
./outer_product_editor 5
```

После запуска откроется окно, разделённое на три области:
- **Верхняя строка** – горизонтальный вектор (N ячеек).
- **Левый столбец** – вертикальный вектор (N ячеек).
- **Основная таблица** – матрица внешнего произведения (N×N ячеек).

Изначально все векторы нулевые, матрица состоит из нулей (серые ячейки).

## Управление

| Действие | Результат |
|----------|-----------|
| **Любая кнопка мыши** по ячейке горизонтального вектора | Инвертировать значение этой ячейки (0↔1) |
| **Любая кнопка мыши** по ячейке вертикального вектора | Инвертировать значение этой ячейки (0↔1) |
| **Alt + V** | Инвертировать **все** ячейки вертикального вектора |
| **Alt + H** | Инвертировать **все** ячейки горизонтального вектора |
| **Esc** | Сбросить все векторы в **0** (матрица также обнуляется) |
| **Ctrl + C** | Завершить работу программы |

После каждого изменения векторов матрица внешнего произведения автоматически пересчитывается и обновляется на экране.

## Архитектура программы (разделение на модули)

### 1. `head.h` – заголовочный файл
- Подключение системных и X11-заголовков.
- Определение константы `CELL_SIZE`.
- Определение типа `Pixel` (если не определён в Xlib).
- Объявление глобальных переменных как `extern` (дисплей, окна, цвета, логические массивы).
- Прототипы всех функций.

### 2. `logic.c` – логический модуль (модель данных)
- Реализация функций:
  - `init_logic()` – выделение памяти под векторы и матрицу, инициализация нулями.
  - `free_logic()` – освобождение памяти.
  - `compute_outer_product()` – пересчёт матрицы по текущим векторам.
  - `invert_horz_cell(idx)`, `invert_vert_cell(idx)` – инверсия одной ячейки.
  - `invert_all_horz()`, `invert_all_vert()` – инверсия всех ячеек вектора.
  - `reset_all()` – обнуление всех векторов.
- Модуль **не содержит вызовов Xlib** (чистая логика). Это позволяет легко тестировать и модифицировать правила вычисления (например, заменить AND на OR).

### 3. `display.c` – графический модуль (представление)
- Определения глобальных переменных.
- Функции:
  - `init_colors()` – инициализация цветов (серый для 0, зелёный для 1).
  - `create_windows()` – создание главного окна, окон векторов и матрицы.
  - `set_cell_color()` – изменение цвета фона окна.
  - `update_display()` – синхронизация отображения всех окон с текущими значениями (вызывается после каждого изменения модели).
  - `run_event_loop()` – главный цикл обработки событий X11:
    - `KeyPress` – отслеживание Ctrl+C, Esc, Alt+V, Alt+H.
    - `ButtonPress` – определение, по какому окну был клик (горизонтальный/вертикальный вектор), и вызов соответствующей функции инверсии.
  - `destroy_windows()` – уничтожение всех окон и освобождение памяти.
- Функция `main()` – парсинг аргументов, открытие дисплея, вызов инициализации и главного цикла.

## Принцип работы (алгоритмическая часть)

### 1. Структура данных
- **Векторы**: два одномерных массива целых чисел (`horz_vals`, `vert_vals`) длины N.
- **Матрица**: двумерный массив `mat_vals` размера N×N, вычисляемый как `vert_vals[i] & horz_vals[j]`.

### 2. Интерактивное редактирование
- При клике по окну горизонтального вектора вызывается `invert_horz_cell(j)`, которая меняет значение в массиве `horz_vals[j]` на противоположное (0→1, 1→0) и затем пересчитывает матрицу (`compute_outer_product()`).
- Аналогично для вертикального вектора.
- Глобальные комбинации (`Alt+V`, `Alt+H`) проходят по всем элементам массива и инвертируют их, после чего также пересчитывают матрицу.
- `Esc` обнуляет оба вектора и пересчитывает матрицу.

### 3. Синхронизация отображения
- После любого изменения модели вызывается `update_display()`, которая для каждого окна:
  - Берёт текущее значение из соответствующего массива (0 или 1).
  - Устанавливает цвет фона окна: серый для 0, зелёный для 1.
- Это обеспечивает полное соответствие между графическим интерфейсом и внутренним состоянием.

### 4. Геометрия окна
- Главное окно имеет размер `(CELL_SIZE + N*CELL_SIZE)` по ширине и высоте.
- Левый верхний угол (0,0) пуст (отступ для размещения вертикального вектора и горизонтального вектора без наложения).
- Горизонтальный вектор: ячейки расположены в строке `y=0`, начиная с `x=CELL_SIZE`.
- Вертикальный вектор: ячейки расположены в столбце `x=0`, начиная с `y=CELL_SIZE`.
- Матрица: ячейки расположены в области `x >= CELL_SIZE`, `y >= CELL_SIZE`.

## Модификации и улучшения

### Изменение логической операции
В функции `compute_outer_product()` в `logic.c` можно заменить `&` на `|` (логическое ИЛИ) или другую бинарную операцию.

### Добавление большего числа цветов
Можно расширить программу, чтобы значения векторов были не бинарными, а, например, трёхзначными (0,1,2) с соответствующей цветовой схемой.

### Сохранение и загрузка состояния
Добавить функции для записи векторов в файл и чтения из файла, с последующим обновлением дисплея.

### Визуализация «единичной» матрицы
При N=5, если установить все единицы в обоих векторах, матрица станет полностью зелёной – наглядная демонстрация.
