# README: Редактор трёхцветных пиктограмм (треугольные ячейки)

## Общее описание

Программа позволяет создавать и редактировать пиктограмму, состоящую из прямоугольной матрицы квадратных клеток. Каждая клетка разделена диагональю на два равнобедренных прямоугольных треугольника. Диагонали в соседних клетках ортогональны (чередуются в шахматном порядке), благодаря чему треугольные ячейки образуют непрерывное мозаичное покрытие.

Каждый треугольник может быть окрашен в один из трёх цветов: **красный (Red)**, **зелёный (Green)** или **синий (Blue)**. Изначально все треугольники красные.

Редактирование производится в графическом окне с помощью мыши и клавиатуры. Все изменения мгновенно отображаются.

Программа написана на языке **C** с использованием библиотеки **Xlib** (X Window System).

## Требования к системе

- Операционная система с X Window System (Linux, BSD, другие UNIX-подобные).
- Установленная библиотека X11 (разработочные файлы).
- Компилятор GCC (или любой другой C-компилятор).

## Компиляция и запуск

### Компиляция из одного файла (исходный вариант)

```bash
gcc -o icon_editor icon_editor.c -lX11 -Wall
```

### Компиляция раздельной сборки (три файла: head.h, logic.c, display.c)

```bash
gcc -o icon_editor display.c logic.c -lX11 -Wall
```

### Запуск

```bash
./icon_editor <число_строк> <число_столбцов> <размер_ячейки_в_пикселях>
```

Пример:

```bash
./icon_editor 10 15 40
```

После запуска откроется окно с матрицей 10×15 клеток, каждая клетка имеет размер 40×40 пикселей. Все треугольники красные.

## Управление

| Действие | Результат |
|----------|-----------|
| **Левая кнопка мыши** по треугольнику | Закрасить треугольник в **красный** цвет |
| **Средняя кнопка мыши** по треугольнику | Закрасить треугольник в **зелёный** цвет |
| **Правая кнопка мыши** по треугольнику | Закрасить треугольник в **синий** цвет |
| **Клавиша R** | Закрасить **все** треугольники в **красный** цвет |
| **Клавиша G** | Закрасить **все** треугольники в **зелёный** цвет |
| **Клавиша B** | Закрасить **все** треугольники в **синий** цвет |
| **Клавиша ESC** | Завершить работу программы |

## Архитектура программы (разделение на модули)

### 1. `head.h` – заголовочный файл
- Подключение системных и X11-заголовков.
- Определение типа `Pixel` (если не определён в Xlib).
- Определение структуры `Cell`, хранящей два цвета треугольников.
- Объявление всех глобальных переменных как `extern`.
- Прототипы функций.

### 2. `logic.c` – логический модуль
- Реализация функций, не связанных напрямую с рисованием:
  - `init_colors()` – инициализация цветов через XColor.
  - `set_all_triangles()` – установка цвета для всех треугольников.
  - `free_cells()` – освобождение динамической памяти.
  - `get_triangle_index()` – определение, по какому треугольнику (0 или 1) произошёл клик.
- Этот модуль оперирует данными (матрица `cells`), но не выполняет вызовы XDraw / XFill.

### 3. `display.c` – графический модуль
- Содержит определения глобальных переменных (память под них).
- Функции рисования:
  - `draw_triangle()` – заливка одного треугольника заданным цветом.
  - `draw_diagonal()` – рисование чёрной линии по диагонали клетки (граница между треугольниками).
  - `draw_cell()` – перерисовка всей клетки (два треугольника + диагональ).
  - `draw_all()` – полная перерисовка всей матрицы.
- Функции инициализации:
  - `create_main_window()` – создание окна, настройка обработки событий.
  - `allocate_cells()` – выделение памяти и заполнение начальными цветами.
- Функция `run_event_loop()` – главный цикл обработки событий X11.
- Функция `main()` – точка входа, парсинг аргументов, вызов инициализации и цикла.

## Принцип работы (алгоритмическая часть)

### 1. Чередование диагоналей

В каждой клетке ориентация диагонали определяется чётностью суммы индексов `(row + col) % 2`:
- **Чётная сумма** – диагональ из левого верхнего в правый нижний угол (символ `/`).
- **Нечётная сумма** – диагональ из правого верхнего в левый нижний угол (символ `\`).

Это обеспечивает, что в любых двух смежных клетках диагонали ортогональны (перпендикулярны), и треугольники на общей границе являются зеркальными отражениями.

### 2. Определение треугольника по координатам мыши

При клике вычисляются координаты клетки `(col, row)` и внутриклеточные координаты `(dx, dy)` (от 0 до `cell_size-1`). В зависимости от ориентации диагонали проверяется, с какой стороны от линии находится точка:
- Для диагонали `/`: если `dy <= dx`, то точка выше диагонали (треугольник 0), иначе ниже (треугольник 1).
- Для диагонали `\`: если `dy <= (cell_size-1 - dx)`, то точка выше диагонали (треугольник 0), иначе ниже (треугольник 1).

### 3. Отрисовка

- Заливка треугольника выполняется через `XFillPolygon` с тремя точками.
- Диагональ рисуется через `XDrawLine` поверх залитых треугольников, чтобы граница была чётко видна.
- После изменения цвета одного треугольника перерисовывается только одна клетка (вызов `draw_cell`). При глобальном изменении перерисовывается всё окно (`draw_all`).
- Главное окно имеет `border_width = 1`, поэтому вокруг каждой клетки видна тонкая чёрная рамка – сетка.

### 4. Обработка событий

- **Expose** – при перекрытии окна или его появлении вызывается полная перерисовка.
- **ButtonPress** – определяются координаты, индекс треугольника, новый цвет (по кнопке) и обновляется ячейка.
- **KeyPress** – клавиши R/G/B меняют все треугольники, ESC – выход.

## Возможные модификации

- Изменение цвета сетки и диагоналей (вместо `BlackPixel` использовать другой цвет).
- Добавление возможности сохранения пиктограммы в файл.
- Изменение размера окна в процессе работы (потребовало бы пересоздания матрицы).
- Поддержка большего числа цветов (например, через палитру).
