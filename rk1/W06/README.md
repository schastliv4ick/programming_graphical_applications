# Редактор трёхцветных пиктограмм (вариант W06)

## Описание

Программа позволяет создавать и редактировать цветные пиктограммы, составленные из квадратных клеток. Каждая клетка разделена диагональю на два прямоугольных треугольника. Диагонали в соседних клетках ортогональны (чередуются направления `/` и `\`).  
Каждый треугольник может быть окрашен в один из трёх цветов: **красный**, **зелёный** или **синий**. Редактирование производится интерактивно с помощью мыши и клавиатуры.

Размеры пиктограммы (количество строк и столбцов) и размер клетки в пикселях задаются через аргументы командной строки.

## Требования

- Операционная система с X Window System (GNU/Linux, BSD и т.п.).
- Компилятор GCC.
- Библиотека Xlib (обычно входит в пакет `libx11-dev` или аналогичный).

## Сборка

Программа состоит из одного исходного файла `icon_editor.c`. Скомпилировать её можно командой:

```bash
gcc -o icon_editor icon_editor.c -lX11
```

Для удобства можно использовать `make` (пример Makefile приведён ниже).

## Запуск

```bash
./icon_editor <строки> <столбцы> <размер_клетки>
```

Все аргументы – целые положительные числа.

- **строки** – количество строк пиктограммы (по вертикали)
- **столбцы** – количество столбцов (по горизонтали)
- **размер_клетки** – размер стороны квадратной клетки в пикселях (например, 50)

### Примеры

```bash
./icon_editor 6 8 50     # 6 строк, 8 столбцов, клетка 50x50 пикселей
./icon_editor 10 10 40   # квадратная матрица 10x10, клетка 40x40
```

## Управление

- **Левая кнопка мыши** по треугольнику → закрасить треугольник в **красный** цвет.
- **Средняя кнопка мыши** → закрасить в **зелёный** цвет.
- **Правая кнопка мыши** → закрасить в **синий** цвет.

- **Клавиша `R` (или `r`)** → установить **красный** цвет для всех треугольников.
- **Клавиша `G` (или `g`)** → установить **зелёный** цвет для всех треугольников.
- **Клавиша `B` (или `b`)** → установить **синий** цвет для всех треугольников.

- **Клавиша `ESC`** → завершить программу.

## Особенности реализации

- Используется библиотека Xlib (чистый X11, без toolkit'ов).
- Каждая клетка не является отдельным окном (в отличие от некоторых аналогов) – всё поле рисуется в одном окне. Это упрощает код и ускоряет перерисовку.
- Направление диагонали в клетке зависит от чётности суммы координат `(row + col)`, что обеспечивает ортогональность диагоналей в соседних клетках.
- Определение треугольника, по которому произошёл клик, вычисляется по координатам мыши внутри клетки с учётом текущей диагонали.
- Все треугольники изначально красные.
- Память под матрицу цветов выделяется динамически в соответствии с заданными размерами.

## Пример Makefile

```makefile
CC = gcc
CFLAGS = -Wall -O2
LIBS = -lX11

all: icon_editor

icon_editor: icon_editor.c
	$(CC) $(CFLAGS) -o icon_editor icon_editor.c $(LIBS)

clean:
	rm -f icon_editor
```
