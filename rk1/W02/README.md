# README: Редактор трёхцветных пиктограмм (квадратные ячейки)

## Общее описание

Программа позволяет создавать и редактировать пиктограмму, состоящую из прямоугольной матрицы **квадратных ячеек**. Каждая ячейка представляет собой отдельное дочернее окно X Window System. Цвет ячейки определяется цветом её фона.

Изначально все ячейки имеют **красный** цвет. Редактирование производится в графическом окне: щелчок по ячейке изменяет её цвет в соответствии с нажатой кнопкой мыши. Также можно закрасить все ячейки одним цветом с помощью клавиатуры. Выход из программы – по нажатию `ESC`.

Программа написана на языке **C** с использованием библиотеки **Xlib**.

## Требования к системе

- Операционная система с X Window System (Linux, BSD, другие UNIX-подобные).
- Установленная библиотека X11 (разработочные файлы).
- Компилятор GCC.

## Компиляция и запуск

### Компиляция из одного файла (исходный вариант)

```bash
gcc -o icon_editor_squares icon_editor_squares.c -lX11 -Wall
```

### Компиляция раздельной сборки (три файла: head.h, logic.c, display.c)

```bash
gcc -o icon_editor_squares display.c logic.c -lX11 -Wall
```

### Запуск

```bash
./icon_editor_squares <число_строк> <число_столбцов> <размер_ячейки_в_пикселях>
```

Пример:

```bash
./icon_editor_squares 8 10 50
```

После запуска откроется окно с матрицей 8×10 ячеек, каждая ячейка имеет размер 50×50 пикселей. Все ячейки красные.

## Управление

| Действие | Результат |
|----------|-----------|
| **Левая кнопка мыши** по ячейке | Закрасить ячейку в **красный** цвет |
| **Средняя кнопка мыши** по ячейке | Закрасить ячейку в **зелёный** цвет |
| **Правая кнопка мыши** по ячейке | Закрасить ячейку в **синий** цвет |
| **Клавиша R** | Закрасить **все** ячейки в **красный** цвет |
| **Клавиша G** | Закрасить **все** ячейки в **зелёный** цвет |
| **Клавиша B** | Закрасить **все** ячейки в **синий** цвет |
| **Клавиша ESC** | Завершить работу программы |

## Архитектура программы (разделение на модули)

### 1. `head.h` – заголовочный файл
- Подключение системных и X11-заголовков.
- Определение типа `Pixel` (если не определён в Xlib).
- Объявление глобальных переменных как `extern` (дисплей, главное окно, матрица дочерних окон, цвета и т.д.).
- Прототипы функций.

### 2. `logic.c` – логический модуль
- Реализация функций:
  - `init_colors()` – инициализация цветов (красный, зелёный, синий) через `XParseColor` и `XAllocColor`.
  - `set_all_cells()` – установка цвета для всех ячеек.
  - `free_cells()` – освобождение памяти, выделенной под массив окон.
- Этот модуль не выполняет непосредственных вызовов Xlib для рисования, но использует XColor (необходимо для получения пиксельных значений).

### 3. `display.c` – графический модуль
- Определения глобальных переменных (память).
- Функции:
  - `create_main_window()` – создание главного окна-контейнера.
  - `create_cells()` – создание матрицы дочерних окон (каждое – отдельная ячейка), установка начального красного цвета фона.
  - `set_cell_color()` – изменение цвета фона конкретного окна с помощью `XSetWindowBackground` и `XClearWindow`.
  - `run_event_loop()` – главный цикл обработки событий:
    - `KeyPress` – клавиши R/G/B меняют все ячейки, ESC – выход.
    - `ButtonPress` – определение, по какому дочернему окну был клик, и смена его цвета в зависимости от кнопки.
  - `free_resources()` – уничтожение всех окон и освобождение памяти.
- Функция `main()` – парсинг аргументов, открытие дисплея, вызов инициализации и цикла.

## Принцип работы (алгоритмическая часть)

### 1. Создание окон
- Главное окно имеет размер `cols * cell_size` на `rows * cell_size`. Оно служит контейнером.
- Для каждой ячейки `(i,j)` создаётся дочернее окно с координатами `x = j * cell_size`, `y = i * cell_size`, размером `cell_size × cell_size`.
- Все дочерние окна отображаются сразу после создания (`XMapWindow`).
- Для каждого дочернего окна устанавливается обработка события `ButtonPressMask`.

### 2. Управление цветом
- Цвет задаётся через фон окна: `XSetWindowBackground(dpy, win, pixel_color)` + `XClearWindow`.
- Начальный цвет – красный (полученный через `XParseColor`).
- При клике мыши по ячейке определяется, какая кнопка нажата, и вызывается `set_cell_color` с соответствующим цветом.

### 3. Обработка событий
- **Expose** для главного окна не требуется, так как дочерние окна перерисовываются автоматически при изменении фона.
- **ButtonPress** – необходимо определить, какое именно дочернее окно было затронуто. Для этого идентификатор окна (`ev.xbutton.window`) сравнивается с элементами массива `cells`.
- **KeyPress** – отслеживаются клавиши R, G, B и ESC.

### 4. Сетка между ячейками
В исходной реализации для видимости сетки можно использовать `border_width = 1` при создании дочерних окон (рамка будет чёрной). Однако в данном варианте акцент на цветовых ячейках, и сетка может быть не обязательна. Для улучшения восприятия можно добавить тонкие рамки (см. раздел «Модификации»).

## Модификации и улучшения

### Добавление видимой сетки
Чтобы между ячейками появились чёрные линии, достаточно в вызове `XCreateSimpleWindow` для каждой ячейки задать `border_width = 1`:
```c
cells[i][j] = XCreateSimpleWindow(dpy, main_win, x, y, cell_size, cell_size, 1,
                                  BlackPixel(dpy, screen), color_red);
```
При этом общий размер главного окна нужно увеличить, так как ширина ячейки становится `cell_size + 2` (по 1 пикселю на каждую границу). Удобнее оставить `border_width = 1`, но при вычислении размеров использовать `total_cell_size = cell_size + 2`. Однако для простоты можно рисовать сетку отдельно через `XDrawLine` в главном окне.

### Поддержка других цветов
Добавление новых цветов требует расширения логики обработки кнопок мыши (например, четвёртая кнопка) и новых глобальных клавиш.

### Сохранение пиктограммы в файл
Можно добавить функцию, которая записывает матрицу цветов в текстовый файл (например, в формате CSV) и восстанавливает из него.
