## Что нужно установить
В Ubuntu необходимо установить компилятор `gcc`, утилиту `make` и библиотеку X11.  
Выполните в терминале:
```bash
sudo apt update
sudo apt install gcc make libx11-dev
```

## Лабораторная работа 1 (буква B)

### Компиляция и запуск программы:
```bash
gcc lab1.c -o lab1 -lX11
./lab1
```

### Важные теоретические моменты (что спрашивали на защите)
- Знать определения **правильного, полуправильного и неправильного замощения**.
- Понимать назначение функций `XCopyArea`, `XSelectInput`, `XSetFunction`, а также что такое маска событий (`mask`). Лучше просто разбираться в коде.
- В некоторых вариантах есть ошибка: цвет меняется не только при нажатии, но и при наведении курсора – нужно исправить.

## Лабораторная работа 2 (буква P)

### Компиляция и запуск
**С Makefile:**
```bash
make
./lab2
```
Для очистки:
```bash
make clean
```
**Вручную:**
```bash
gcc -Wall -O2 lab2_control.c lab2_display.c lab2_geom.c -lX11 -o lab2
./lab2
```
*Убедитесь, что все исходные файлы (`lab2.h`, `lab2_control.c`, `lab2_display.c`, `lab2_geom.c`) находятся в одной папке.*

### Важные замечания по коду и теории
- Если не работает перерисовка по `Ctrl+L` – ищите ошибку в функции `rekey` в файле `lab2_control.c`.
- Попробуйте закрасить фон и растянуть окно. Если раскраска «ломается» (грани не перерисовываются при изменении размера), это тоже нужно починить.
- По умолчанию окно должно быть квадратным с изображением графа. Чтобы сделать его квадратным, в файле `lab2_display.c` после объявления переменных `w` и `h` приравняйте `h = w`.
- Вопросы по теории графов:  
  * Когда плоский граф можно раскрасить всего двумя цветами так, чтобы соседние грани были разного цвета? (Ответ: когда все вершины имеют чётную степень.) 
  * Теорема о четырёх красках.
