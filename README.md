## Компиляция программы (на примере lab2)

Для компиляции программы, использующей библиотеку X11, проще всего создать **Makefile**. Это автоматизирует процесс сборки и позволит легко перекомпилировать проект после изменений.

### Как использовать Makefile

1. Откройте терминал (командную строку) в папке с проектом.
2. Убедитесь, что у вас установлен компилятор `gcc` и библиотека X11 (обычно уже есть в большинстве дистрибутивов Linux).
3. Введите команду:
   ```bash
   make
   ```
   Будут скомпилированы все объектные файлы и создан исполняемый файл `lab2`.
4. Чтобы запустить программу, наберите:
   ```bash
   ./lab2
   ```
5. Если нужно удалить объектные файлы и исполняемый файл, выполните:
   ```bash
   make clean
   ```

### Альтернативный способ (компиляция вручную)

Если вы не хотите использовать Makefile, можно скомпилировать программу одной командой:

```bash
gcc -Wall -O2 lab2_control.c lab2_display.c lab2_geom.c -lX11 -o lab2
```

Затем запустить `./lab2`.

### Примечания

- Убедитесь, что в системе установлены заголовочные файлы Xlib (пакет `libx11-dev` в Debian/Ubuntu, `libX11-devel` в Fedora/CentOS).
- При изменении исходных файлов повторно выполните `make` для пересборки.
- Если вы переименовали исполняемый файл, укажите соответствующее имя в `TARGET`.
